名家名著

始于汉代，成熟于唐代，沿用到清朝的书籍分类：

  • 经：儒家经典，包括《易》（《周易》）、《书》（《尚书》）、《诗》（《诗经》）、《礼》（如《周礼》、《仪礼》、《礼记》）、《春秋》（《春秋左传》、《春秋公羊传》、《春秋穀梁传》）以及《论语》、《孟子》等；
  • 史：历史文献，包括司马迁的《史记》、班固的《汉书》、范晔的《后汉书》、陈寿的《三国志》等正史，还有《资治通鉴》等编年体通史；
  • 子：除儒家经典之外的各种学术著作，如道家的老子《道德经》、庄子《庄子》，墨家的《墨子》，法家的《韩非子》，兵家的《孙子兵法》，医家的《黄帝内经》等
  • 集：文学作品的汇编，包括诗歌、散文、辞赋、小说等，如屈原的《楚辞》，陶渊明的《陶渊明集》，李白的《李太白全集》，杜甫的《杜工部集》，以及《全唐诗》、《全宋词》等大型总集；

周：西周、东周、春秋、战国

四书：

  1. 《论语》，春秋，孔子及其弟子；
  2. 《孟子》，战国，孟子及其弟子；
  3. 《大学》，战国，曾子，《礼记》中的一篇，在宋代独立成书；
  4. 《中庸》，战国，相传孔子的孙子子思所作，在宋代独立成书；

五经：

  1. 《诗经》：西周至春秋中期，周代官方收集整理的民间歌谣和贵族诗歌；
  2. 《尚书》：或称《书经》，上古至春秋，从尧舜到秦穆公时期的帝王诏令、训诫和史事；

音韵学

韵书

韵书是中国古代按调和韵编排的字典，记录了每个字的意义之外，还用反切记录了它的读音。

  1. 《声类》，李登， 魏，已失传；
  2. 《韵集》，吕静，晋，已失传；
  3. 《切韵》，陆法言，公元 601，隋，分 193 韵部；
  4. 《唐韵》，孙愐，公元 732，唐，分 195 韵部，修订《切韵》，已失传；
  5. 《广韵》，陈彭年、邱雍，公元 1007-1008，北宋，分 206 韵部，全称《大宋重修广韵》，修订《唐韵》；
  6. 《集韵》，丁度，公元 1037，北宋，分 206 韵部，修订《广韵》；

《切韵》到了 1947 年才被发现全本，而《集韵》传世的抄本错误很多，也受宋代语音变化影响，因此音韵学研究以《广韵》为主。

Unihan 数据中 Unihan_DictionaryIndices.txt 的 kSBGY 属性记录了汉字在《宋本广韵》的页码位置，Unihan_Readings.txt 中的 kFanqie 属性记录了汉字的反切读音。

平水韵

简化自《广韵》，用于诗歌和散文的押韵，分 106 韵部，编纂于金朝。

  1. 《新刊韵略》，王文郁，公元 1223，金，分 106 韵部，已失传；
  2. 《壬子新刊礼部韵略》，刘渊，原籍江北平水（今山西临汾），公元 1252，南宋，分 107 韵部，基本跟《新刊韵略》一样，已失传；
  3. 《韵府群玉》，阴时夫，元，分 106 韵部；
  4. 《佩文韵府》、《佩文诗韵》、《诗韵合璧》，清；

其它

  1. 《中原音韵》，周德清，元，分 19 韵部，适合戏曲和散曲创作；
  2. 《词林正韵》，戈载，清，简化平水韵为 19 韵部，适合词牌创作；
  3. 十三辙，用于北方说唱艺术如京剧，分 13 韵部；
  4. 《中华新韵》，中华诗词学会，2005，分 14 韵部，尊重音理，平衡了传统与现实，在诗词界接受度较高，被认为是较为成功的现代诗韵；
  5. 《中华通韵》，中华诗词学会，2019，分 15 韵部，作为国家标准，意图统一和规范当代诗韵，但被批评为机械地依据汉语拼音书写形式，违背了音位学和听觉感知原则，在学术界和诗词创作圈引发较大争议，认为其科学性和实用性不如《中华新韵》；

什么是编程？

计算机组成

Computer 一词最早出现于 1613 年，指人类「计算员」，使用计算器做算术运算，直到 20 世纪中期，才被用于指「可编程的数字电子计算机」，简称「计算机」，中文里也被翻译成「电脑」。

一台计算机由硬件(hardware)和软件(software)组成，「硬件」指直接可见可触摸的设备，「软件」指存储、运行在硬件中的不能直接可见可触摸的数据和程序。

硬件

  • 计算和控制：最典型的是中央处理器（CPU，Central Processing Unit)，主流的 CPU 家族有 X86、ARM。CPU 主要有如下元件组成：
    • 算术逻辑单元(ALU, Arithmetic Logic Unit)：负责算术和逻辑运算，包含加法器、乘法器、除法器等电路；
    • 控制单元(CU, Control Unit)：负责解释指令并控制其它部件执行；
    • 寄存器(Registers)：存储 CPU 当前运行用到的指令、数据；
    • 缓存(Cache)：在寄存器和内存之间缓存指令和数据；
  • 存储器
    • 主存储器、内存(memory)：一般指断电即丢失数据的随机访问存储（ RAM，Random Access Memory）；
    • 辅存储器、外存(storage)：包括软盘、硬盘、光盘等，断电不会丢失数据的存储设备；
  • 输入设备(Input Device)：键盘、鼠标、麦克风、扫描仪、网卡等，用于外界输入数据给计算机处理；
  • 输出设备(Output Device)：屏幕、扬声器、打印机、指示灯、网卡等，用于计算机输出结果给外界；
  • 总线(Bus)：连接各个设备的线缆，传输数据和控制信号；

软件

  • 代码 (Code)：用编程语言写成的文本，人可以阅读和修改；
  • 操作系统(OS, Operating System)：狭义指直接管理硬件的操作系统内核(Kernel) ，广义指硬件上的基础软件。主流的 OS 有 Windows、Linux、BSD、macOS、Android、iOS；
  • 程序(Program)：也称为应用(application)，可供操作系统运行；
    • Shell：指操作系统提供的最基本的人机交互界面应用，用于输入命令、查看结果，例如 Linux、BSD 上的 sh 命令行工具，Windows、macOS 上提供的桌面、任务栏、程序列表、文件管理器等图形工具；
    • 编辑器(Editor)：一般指文本编辑器(Text Editor)，可以用来编辑代码保存到外存里；
    • 编译器(Compiler)、汇编器(Assembler)、链接器(Linker)、加载器(Loader)：
      • 编译器：指将高级编程语言翻译到汇编语言的程序；
      • 汇编器：指将汇编语言翻译到机器码的程序；
      • 链接器：指将多个机器指令的程序片断合并到一起的程序；
      • 加载器：指操作系统用来加载外存上的程序进入内存的程序；
    • 解释器(Interpreter)：指直接运行代码，不用显式的经过编译、汇编、链接、加载四个阶段的一类程序；

高级编程语言和汇编语言

下面是三种语言定义函数的语法，以及机器码的十六进制表示，Python 和 C 都属于高级语言。

	return n * n

编程语言的实现和编写流程

编程语言有三种实现方式：

  1. 显式编译后执行，如 C、C++ 语言；
  2. 直接解释执行，如 Python；
  3. 编译到中间形式（还没到机器码）再解释执行，如 Java。

不同的编程语言实现，其编写程序的流程也不一样，也分成三种：

  1. 「编辑 → 编译 → 执行」循环，典型如 C/C++ 和 Java。
#include <iostream>

int main(int argc, char** argv)
{
	std::cout << "Hello world!\n";
	return EXIT_SUCCESS;
}

$ g++ -o hello hello.cpp
$ ./hello
Hello world!

public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello world!");
  }
}

$ javac HelloWorld.java
$ java HelloWorld
Hello world!
  1. 「编辑 → 执行」循环，典型如 Python。Java 也支持这种方式用于简单程序。
#!/usr/bin/env python3

print("Hello world!")

$ python3 hello.py
Hello world!

# 或者
$ chmod a+rx hello.py
$ ./hello.py
Hello world!

# Java 也可以不用显式编译
$ java HelloWorld.java
Hello world!
  1. 「读取 → 执行」循环，也称为 REPL(read-evaluate-print-loop，print 指输出表达式的值)，典型如 Lisp 和 Smalltalk。Python 和 Java 也支持这种方式用于简单测试。
$ sbcl

Hello world!
NIL

$ python3
>>> print("Hello world!")
Hello world!

$ jshell
jshell> System.out.println("Hello world!")
Hello world!

从 Java 的例子可以看到「编程语言」和「编程语言的实现」是两个不同的概念，前者指语法和语义，后者指编译、执行形式。

编程风格的分类

编程风格(style)也常被称为编程范式(paradigm)，指如何组织数据和代码。一种编程语言可以以某种风格为主，同时支持多种风格，常见的编程风格如下：

  • 命令式(Imperative)，也经常狭义指代「过程式」，明确指定计算机一步步「怎么做(how)」；
    • 过程式(procedural)，如 Fortran、Basic、Pascal、C；
    • 面向对象式(object-oriented)
      • 基于类(class-based)，如 C++、Java；
      • 基于原型(prototype-based)，如 JavaScript、Io；
  • 声明式(declarative)，只给出「要做什么(what)」，让计算机（实际是设计好的算法）给出「怎么做」的具体步骤；
    • 函数式(functional)，如 Haskell、OCaml；
    • 逻辑式(logical)，如 Prolog；
    • 响应式(reactive)，如 Verilog，Elm；

截止 2024 年，主流使用的编程语言以过程式、面向对象式为主。

基本语法

编程语言很多，语法各异，但都有一些类似的基本语法。

数据类型

从 CPU 的机器指令视角看，数据类型只有两大类：

  • 标量(scalar)
    • 整数
      • byte(字节)：存储单位，一个字节包含 8 比特(bit)，分有符号字节，范围 0 ~ 255，和无符号字节，范围 -128 ~ 127；
      • word(字)：计算单位，表示 CPU 一条整数指令能计算的数据长度，在 32 位 CPU 中是 4 字节（也就是 32 比特)，在 64 位 CPU 中是 8 字节（也就是 64 比特）；
    • 浮点数
      • float(单精度浮点数)：4 字节；
      • double(双精度浮点数)：8 字节；
      • long double(扩展精度浮点数)：12 或 16 字节；
  • 向量(vector)：也就是数组(array)，在内存中连续存储的多个标量；

从编程语言的语法角度看，使用变量和常量指代数据，数据类型表现的更多样，以 C/C++ 为例：

  • 整数：char, short, int, long, long long, int8_t, int16_t, int32_t, int64_t 等，以及对应的 无符号(unsigned)版本；
    • char 既表示字节，也表示 ASCII 字符，区分只在输入、输出上；
    • 布尔类型 (_Bool, bool) 也是整数，但只有两个取值 true 和 false；
    • 指针类型，本质也是个整数，用来表示内存地址；
  • 浮点数：float, double, long double，符合浮点算术标准 IEEE 754；

由于长度限制，整数能表达的范围是有限的，N 比特能表达 2N 个整数，所以无符号整数范围是 0 ~ 2N - 1，有符号整数范围是 -2N ~ 2N - 1。因为有长度限制，运算时会发生溢出回绕，例如 signed char 的 127 + 1 = -128， -128 - 1 = 127，unsigned char 的 255 + 1 = 0，0 - 1 = 255。有的编程语言如 Python 的默认整数类型是利用多个机器整数拼起来的，可以表达任意小和任意大的整数，不会发生溢出回绕。

类似的，由于长度限制，浮点数能表达的范围和精度是有限的，特别是精度问题，浮点数不应该用 "==" 判断相等，例如 0.1 + 0.2 == 0.3 的结果是 false，并不相等。有的语言支持分数类型，可以用来做精确的四则运算，结果依然为分数。

除了整数和浮点数两种基本类型，大多编程语言还有以下复合类型：

道路通行规定

  1. 四项基本原则
    1. 右侧通行；
    2. 分道通行：机动车道、非机动车道、人行道；
    3. 多车道：左为快、右为慢，不要侵占公交专用车道；
    4. 单车道：中间通行；
  2. 车辆灯光
    1. 灯光控制拔杆
      1. 示廓灯（示宽灯、行车灯） + 后位灯
      2. 前照灯：分为近光灯、远光灯；没有交通信号灯的路口要交替使用近光、远光灯来示意；会车 150m 外不能用远光灯；
      3. 转向灯
    2. 单独开关
      1. 危险报警闪光灯(双闪灯)
      2. 雾灯：分为前雾灯和后雾灯​​，先打开​​示廓灯​​或​​近光灯​​，然后才能开启雾灯（这是电路设计逻辑）；
    3. 自动打开
      1. 制动灯（刹车灯）
      2. 倒车灯：自动打开
  3. 设置警告标置：50 ~ 100m，雨天 >100m
  4. 交叉路口通行
    1. 拐弯让直行；
    2. 支路让主路；
    3. 无信号灯或无方向指示信号灯路口：
      1. 右方道路来车先行；
      2. 对向行驶：右转让左转，因为左转路径复杂；
    4. 左转时要从路口中心左侧绕行，以避免与对向左转车相撞；
  5. 环岛
    1. 进环岛：不打转向灯（因为右转是出环岛，左转是逆行）；
    2. 出环岛：打右转向灯；
    3. 岛外车让岛内车先行；

跟车与限制超车

  1. 超车：
    1. 提前3-5秒开启左转向灯；
    2. 观察内后视镜、左后视镜，转头快速扫视车辆B柱后方的区域；

    5. 确认拉开充足距离后，右转向灯，驶回原车道；
  2. 会车
    1. 减速靠右；
    2. 遇到前方障碍时的处理，无障碍一方先行，但已经驶入障碍区的车先行；

      1. 上坡：上坡车先行，但已经在下坡的车先行；
      2. 盘山路：外侧车先行；
    4. 相距 150m 外改用近光灯；
    5. 辅路让主路；
  3. 掉头与倒车

绳结简介

基础结

  1. Overhand knot：单结，反手结，交腕结。最简单的单股止索结，难解开。
    1. Figure-eight knot：八字结，较易解开的止索结。
    2. Slip knot：活结，活动端折弯的单结。
    3. Noose knot：套索，最简单的滑结(running knot)，固定端折弯的单结。
  2. Half knot：半结，最简单的捆绑结，平结的第一步（其实拓扑上就是单结)。
    1. Square knot：平结(方结)，最常用的捆绑结。
  3. Half hitch: 半结，最简单的拴结(其实拓扑上就是单结）。
  4. Sheet bend：单编结，最常用的连接结，可以连接不同粗细的绳子。

绳结的考量点：

  1. 用途：
    1. 捆绑(binding) ：Boa knot, Miller's knot, Reef knot
    2. 连接(bend) ：Simple Simon double bend, Simple Simon symmetric bend, Zeppelin bend, Butterfly bend，Double/Triple fisherman's knot
    3. 拴系(hitch) ：Round turn and two half-hitches，Boom Hitch, EStar hitch, Midshipman's hitch, Tumble hitch, Siberian hitch
    4. 抓结(friction hitch) ：Prusik hitch
    5. 防脱(stopper) ： EStar stopper knot, Double figure-eight knot(Stevedore knot), Double overhand knot
    6. 绳环(loop) ：Butterfly loop, Double butterfly loop, Figure-eight loop/Figure-nine loop, Double figure-eight loop, Zeppelin loop
    7. 滑结(running knot) ： Running bowline
    8. 盘绳(coil)：Butterfly coil, Chain Sinnet
  2. 安全：是否易打滑、翻转，能承受多大负荷，适用多粗多细、什么材料的绳子，是否适用光滑、弹性的绳子，是否易磨损绳子，结的大小(影响卡塞），结构是否对称（影响受力均匀）。
  3. 易用：步骤是不是够少，是不是易错，是不是容易解开，能不能单手操作，能否在受力时打结、解开、调整，易于检查是否正确。

绳子种类：

  1. 天然纤维绳
    1. 棉绳
    2. 麻绳
    3. 棕绳
    4. 丝绳
  2. 合成纤维绳
    1. 聚丙烯绳，Polypropylene，丙纶，聚丙烯纤维，能浮水，低强度，不耐磨。
    2. 尼龙绳，Nylon，绵纶，耐纶，聚酰胺纤维，高弹性，可用作动力绳和静力绳（需特殊工艺降低弹性）。
    3. 涤纶绳，Polyester，聚酯纤维，低弹性，常用作静力绳。
    4. 芳纶绳，Aramid，如凯夫拉(Kevlar)，芳酰胺纤维，高强度但很脆，耐高温。
    5. 聚乙烯绳，Polyethylene，如迪尼玛(Dyneema)，能浮水，高强度，不耐磨，不耐高温。

注意事项：

  1. 不要将 Reef knot 用于连接两条绳子！！！
  2. 不要单独使用 Clove hitch！！！
  3. 不要使用不安全的标准 Bowline，最好避开 Bowline 家族，变种太多，难以检查，很多不安全。
  4. 止索结要缠绕在主绳上，不能只打在绳头自身。
  5. 相比单重版本的绳结，为了安全，倾向使用双重甚至三重版本。
  6. 滑结一般不要用在人身上，以免过于勒紧而发生危险——Noose 和 Poacher's knot 都很危险。

Stopper knot：止索结

防止绳子从孔隙中脱落。

  1. Overhand knot: 单结，反手结，交腕结，难解开。
    1. Double overhand knot：双反手结，不易松动，难解开。
    2. Double fisherman's knot，也可用于止索结。
    3. Strangle knot，扼结，也可用于止索结。
  2. Figure-eight knot: 八字结，比反手结更大、易解，但易松开。
    1. Double figure-eight knot, Stevedore knot：双八字结，码头工人结/装卸工结。更大，易解。
  3. Ashley's stopper knot, Oysterman's stopper knot: 阿什利止索结，牡蛎渔夫止索结，不易松动，比双反手结和双八字结大。
  4. EStar stopper knot，由 Evans Starzinger 描述，源自 EStar hitch，更适合光滑的绳索。重负载后难解开。
  5. Monkey's fist：猴拳结

Binding knot：捆绑结

用一根绳子捆绑物体。有的 binding knot 也可用作 hitch knot。

  1. Friction knots: 靠绳子的摩擦力固定，摩擦力由小到大：
    1. Miller's knot, bag knot：磨坊主结，米勒结，常用来绑袋子口，易绑也易解。
    2. Strangle knot: 扼结，简单，常用来绑袋子口和缚扎鞭子末端。没 constrictor knot 可靠。
    3. Constrictor knot：紧缚结，可能难解开。
    4. Boa knot：蟒蛇结，Strangle knot(扼结）和Double constrictor knot(双重紧缚结）的结合，更适合捆绑物体末端。可能更难解开。
  2. Knotted-ends knots: 靠绳子末端打结来固定
    1. Reef/Square knot: 平结、方结，由两个半结组成。狭义的方结指没有滑结的版本，而平结指一头是滑结的版本。因拉扯自由端时容易翻转散开，因此千万不要用作连接结！！！
      1. Surgeon's knot：外科结，适合细绳，打第一个结时多绕几圈的平结改良版
      2. Shoelace knot(bow knot)：鞋带结，领结，两头都有滑结的平结
      3. Thief knot：贼结
      4. Granny knot：奶奶结
      5. Grief knot(what knot)：悲伤结（莫名其妙结）

Lashing knot：绑扎结

用一根绳子将多根物体固定在一起。

Bend (knot)：连接结

也叫 joining knot，连接两条绳子。

  1. 适用不同粗细绳子
    1. Sheet bend，Double sheet bend： 单编结，双编结，织工结(weaver's knot, weaver's hitch)，易用，注意单编结在光滑或受力不均时可能滑脱。
      1. Simple Simon under bend：西蒙下接结，适用于不同粗细绳子和光滑的合成绳索，优于 sheet bend，但未成为主流标准。注意 simple Simon over bend 不安全。
      2. Simple Simon double bend
      3. Simple Simon symmetric bend：有点像 Reever knot
  2. 适用相同粗细绳子
    1. 两个反手结互锁的结构
      1. Zeppelin bend: 齐柏林结，飞艇结，非常易解，比猎人结更好。
      2. Butterfly bend：蝴蝶连接结，易解，但易错。
      3. Hunter's bend: 猎人结，很像 Zeppelin bend，多一步缠绕，受力后难解。
      4. Ashley's bend: 阿什利结，打法复杂，受力后很难解。
    2. Fisherman's knot：渔夫结，渔人结，易绑，难解。
      1. Double fisherman's knot (grapevine knot)：双渔夫结，葡萄藤结，极难解，攀岩行业标准。
      2. Triple fisherman's knot：三重渔夫结，极高安全性。
    3. Water knot: 水结，扁带结，专用于扁平织带，不可用于圆绳，受力后难解。易滑动，不太安全。
    4. Reever Knot：由登山家设计，旨在提供与双渔人结同等的可靠性，但承重后更容易解开。测试报告显示它表现良好，但不如双渔人结或齐柏林结那样普及。
    5. Flemish bend, figure-eight bend, double figure-eight bend：双重八字结，安全，但体积大，打结麻烦，受力后难解开。
    6. Carrick bend：卡里克结，金钱结，特别适合非常粗重的绳索，易解，但易错。
    7. Offset overhand bend, thumb bend：偏置反手结，拇指结。非常容易系，但可能比较难解。因其体积小和偏置的特性，不容易被岩缝卡住，容易绕边缘移动，所以在沿绳速降方面很受欢迎。在高负载下可能翻转而不安全。注意不要用 offset figure-eight bend，它在低负载下就会翻转而不安全。
    8. Blood knot：血结，常用于钓鱼时连接单丝尼龙线，很难解。

Hitch (knot)：拴系结

拴在柱子、缆绳、环上来固定。

  1. Clove hitch： 丁香结、双套结，由两个 single hitches 叠加组成，易滑动，易松动，常用来组成其它绳结。！！！不要单独使用！！！。 相关的绳结类型：
    1. Constrictor knot
    2. Miller's knot
    3. Strangle knot
    4. Ground-line hitch
  2. Round turn and two half-hitches：旋圆双半结，双绕双半结，简单可靠。
  3. Buntline hitch：收帆索结（将帆底部的收帆索固定到帆桁上），无法在受力状态下打结，稳固但难解，因此常用收帆索滑结(slipped buntline hitch)。系领带的 Four-in-hand knot 其实就是 buntline hitch。
    1. EStar hitch：改进 buntline hitch，更适合光滑的绳索。
    2. Gnat hitch：蚊结，简单可靠，比 buntline hitch 易结也易解，适合轻负载，必须维持拉力，一旦松弛即松脱。
  4. Boom hitch：吊杆结，即使在光滑的钢管上也很难滑动。比 Buntline hitch 易解。
  5. Taut-line hitch：紧绳结，营钉结，适合帐篷拉线，不适合长期固定。有三种变体，其中 见习船员结(Midshipman's hitch) 是最安全的，但在重力下更难调节。
  6. Timber hitch：曳木结，专为拖曳设计，松开负载则绳结散开。
  7. Cow hitch：牛结，除非对绳索的两个固定部分施加相等的张力，否则可能会失效。易滑动。
    1. Bale sling hitch：包索吊结，把 cow hitch 的两个自由端相连，也即用绳环系到杆状物。易滑动。
  8. 临时拴系，一拉即解快速释放，注意强度不高，可能不稳定。
    1. Tumble hitch：翻转结，更好的强盗结，最安全的快速释放绳结。
    2. Highwayman's hitch：强盗结（因为逃跑时要快速解开马的缰绳），容易翻转滑脱。
    3. Siberian hitch：西伯利亚结，方便戴着手套也能系。
    4. Halter hitch：缰绳结，用来把动物系在柱子或环上。其实是个套索，为了避免动物拉开，可以把套索的自由端穿过套索的弯。没有 Siberian hitch 牢固。
    5. Mooring hitch：系泊结。注意不适用于长期停泊。

Friction hitch (knot)：摩擦结、抓结

也叫 slide & grip hitch，系在另一根绳上用来助力上升、下降。

  1. 可双向受力
    1. Prusik：普鲁士结。
  2. 只可单向受力
    1. Klemheist knot, Machard knot, French prusik: 克莱姆海斯特结，马沙尔结，法式普鲁士结。
    2. Distel hitch：迪斯泰尔结。
    3. Bachmann knot： 巴赫曼结。

Loop knot：环结

形成固定大小的圈套。

  1. 在绳子中间结环
    1. Butterfly loop： 蝴蝶结，lineman's loop, butterfly knot, alpine butterfly knot, lineman's rider，在绳子中间结一个环。可能打错成 false butterfly 而容易发生滑动。双环独立受力。
      1. Double Butterfly Loop：双环蝴蝶结。
    2. Figure-eight loop, Figure-eight follow-through, Flemish loop：八字环结，难解开
      1. Figure-nine loop：九字环结，比八字环结使用的绳索更多且更为笨重，但它在某种程度上强度更高，也更不容易卡住。
      2. Double figure-eight loop：形成双环。
    3. Farmer's loop：农夫结
    4. Bowline on a bight: 绳耳上之称人结，形成双环，当仅对一个绳环施加负载时，绳结可能会滑脱。易滑动。
      1. Karash double loop：卡拉什双环。
    5. Spanish bowline：西班牙布林结，形成双环。双环共享受力，只适合一同受力，可能滑动失衡。
      1. Fireman's chair knot：消防员椅结，在收紧前可调节。现代已很少使用。
  2. 在绳子末端结环
    1. Zeppelin loop， zeppelin eye knot：齐柏林环，能用于蹦极绳上，易解开。
    2. Angler's loop, perfection loop：钓鱼者结，完美环结，能用于蹦极绳上，很难解开。
      1. Double dragon loop，双龙环，跟 Double angler's loop 很相似。适合高模量和光滑的绳索，如迪尼玛绳，相比bowline 更不易松动。即使在承受重负荷后，也能相当容易地解开。
    3. Trident loop：强度介于 bowline 和 figure-eight loop 之间。
    4. Bowline：布林结，称人结，单套结，广泛使用，被誉为「绳结之王」，与 sheet bend 和 clove hitch 一起都是最基本的绳结（但单独用都不安全！）。Bow line 指系在船头拉住方帆边缘以让帆兜住风的绳子，也指在船头拉住岸边泊位的绳子。Bowline 易结易解，简单可靠，但在不受载荷（或承受周期性载荷）时容易松动，受到侧向拉力时会滑动，以及在某些情况下绳结的环部会发生翻转形成致命的活结。Bowline 其实是在绳子折弯处的 sheet bend。
      1. Cowboy bowline, left-hand bowline，布林结的稍微安全版本，但也有同样的问题，不建议用于性命攸关的场合。
      2. Double bowline，round turn bowline, 双布林结。与八字结相比，双布林结的优点是在坠落受力后更容易解开，因此被那些多次在先锋攀登中坠落之后难以解开八字结的运动攀岩者所使用。双布林结的缺点是安全性不如八字结，打结所需时间更长，且检查起来也没那么容易。
      3. Water bowline：适合潮湿环境。
      4. Yosemite bowline：存在系错以及在固定前就发生翻转的风险。
      5. Portuguese bowline: 葡萄牙式布林结，形成可调节大小的双环。
      6. Triple bowling：形成三环。

Running knot：滑结

活结(slip knot)和套索(noose)都基于带滑段的反手结（a slipped overhand knot)，拉动工作端(自由端)打开活结，拉动固定端打开套索。

  1. Noose：套索，最简单，但不可靠。套住物体后继续拉主绳时，圈无限缩小，直至勒紧或切断。
    1. Hangman's knot：绞索
  2. Running bowline：滑布林结，通用首选，很可靠，易解开。套住物体后继续拉主绳时，圈收紧到一定程度后自锁，不再缩小 → 安全！
  3. Poacher's knot：偷猎者结，极可靠，很难解开。套住物体后继续拉主绳时，圈持续缩小，缓慢勒紧。
    1. Scaffold knot： 三圈版 Poacher，更难解开。
  4. Honda knot

Coil：绳卷

收纳绳索便于存放。

  1. Butterfly coil：蝴蝶绳卷，也适用于短小的线，如耳机线。
  2. Mountaineer's coil：登山者盘绳卷，比蝴蝶结卷更容易产生扭曲和缠绕。
  3. Chain Sinnet: 链式绳卷，方便收纳、洗涤。
